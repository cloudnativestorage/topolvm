# TDE Provider: GCP Cloud KMS

**Implements:** `keyprovider.KeyProvider`.
**Acceptance:** passes `providertest.Run` against a real Cloud KMS symmetric key (or the KMS emulator) in CI, plus the secret-leak grep.
**File:** `internal/keyprovider/gcpkms.go`, tests `gcpkms_test.go` (build tag `kms_gcp`).

---

## 1. SDK

`cloud.google.com/go/kms/apiv1` (`kms.NewKeyManagementClient`).

## 2. Key difference from AWS

Cloud KMS has no `GenerateDataKey` for symmetric keys. The DEK is generated by the driver (or by `GenerateRandomBytes`) and then wrapped with `Encrypt`. AAD binds the volume.

| Interface method | Cloud KMS call | Notes |
|---|---|---|
| `GenerateDEK` | `GenerateRandomBytes` (or `crypto/rand`) then `Encrypt` | `AdditionalAuthenticatedData = volumeID`. |
| `Unwrap` | `Decrypt` | same `AdditionalAuthenticatedData`. |
| `Rewrap` | `Decrypt` then `Encrypt` (in-process) OR rely on version rotation | See rotation note. |
| `KEKVersion` | `GetCryptoKey` -> `Primary.Name` | the primary `CryptoKeyVersion` resource name. |

`keyRef` is the CryptoKey resource: `projects/P/locations/L/keyRings/R/cryptoKeys/K`.

## 3. Rotation semantics

Cloud KMS rotates by adding a new `CryptoKeyVersion` and moving the primary pointer. `Decrypt` auto-selects the version that produced the ciphertext, so old blobs stay valid. Therefore:

- `KEKVersion` returns the primary version resource name (changes when the key rotates).
- `Rewrap` to adopt the new primary requires `Decrypt` then `Encrypt`. This briefly exposes the DEK plaintext in the driver process; hold it in a `SecretBuf` and zeroize. If avoiding any in-process plaintext is required, treat AWS/Vault as the rewrap-capable providers and let GCP rely on transparent version selection (reconciler updates only `KEKVersion` metadata without re-encrypting). Make this behavior a config toggle `rewrapMode: reencrypt|metadata-only`.

## 4. Auth

Workload Identity on GKE: bind the K8s service accounts to a Google service account with `roles/cloudkms.cryptoKeyEncrypterDecrypter` on the key. ADC consumes the projected token. Node SA gets encrypt/decrypt only.

## 6. Skeleton

```go
type gcpProvider struct{ c *kms.KeyManagementClient; defaultKey, rewrapMode string }

func (p *gcpProvider) GenerateDEK(ctx context.Context, o keyprovider.KeyOpts) (keyprovider.SecretBuf, keyprovider.WrappedKey, error) {
	key := orDefault(o.KeyRef, p.defaultKey)
	dek := crypt.NewSecretBuf(32)
	if _, err := rand.Read(dek.Bytes()); err != nil { return nil, keyprovider.WrappedKey{}, err }
	resp, err := p.c.Encrypt(ctx, &kmspb.EncryptRequest{
		Name: key, Plaintext: dek.Bytes(),
		AdditionalAuthenticatedData: []byte(o.VolumeID),
	})
	if err != nil { dek.Destroy(); return nil, keyprovider.WrappedKey{}, err }
	ver, _ := p.primaryVersion(ctx, key)
	return dek, keyprovider.WrappedKey{Ciphertext: resp.Ciphertext, KeyRef: key, KEKVersion: ver, Provider: "gcp-kms"}, nil
}
// Unwrap -> Decrypt(Ciphertext, AAD=volumeID) -> SecretBuf.
// Rewrap -> metadata-only: refresh KEKVersion; reencrypt: Decrypt then Encrypt under current primary.
// KEKVersion -> GetCryptoKey().Primary.Name.
func (p *gcpProvider) BindsContext() bool { return true }   // AAD
```

## 7. Tests

`gcpkms_test.go` (tag `kms_gcp`): `providertest.Run`. CI uses the Cloud KMS emulator (`gcloud beta emulators kms`) or a sandbox project. Untagged unit test asserts AAD is set and the DEK is 32 bytes and zeroized on Encrypt error.
