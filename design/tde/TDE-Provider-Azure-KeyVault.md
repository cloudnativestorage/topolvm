# TDE Provider: Azure Key Vault / Managed HSM

**Implements:** `keyprovider.KeyProvider`.
**Acceptance:** passes `providertest.Run` against a real Key Vault key (or Managed HSM) in CI, plus the secret-leak grep. Note the context-binding caveat in Section 3.
**File:** `internal/keyprovider/azurekv.go`, tests `azurekv_test.go` (build tag `kms_azure`).

---

## 1. SDK

`github.com/Azure/azure-sdk-for-go/sdk/security/keyvault/azkeys` (key wrap/unwrap), `github.com/Azure/azure-sdk-for-go/sdk/azidentity` (auth).

## 2. Model: wrap a locally generated DEK

Key Vault exposes `WrapKey`/`UnwrapKey` on a stored key. The DEK is generated by the driver and wrapped by the Key Vault key (the KEK never leaves the vault / HSM).

| Interface method | Key Vault op | Notes |
|---|---|---|
| `GenerateDEK` | `crypto/rand` then `WrapKey` | RSA: `RSA-OAEP-256`. Managed HSM AES: `A256KW`. |
| `Unwrap` | `UnwrapKey` | returns the DEK bytes. |
| `Rewrap` | `UnwrapKey` then `WrapKey` under current key version | brief in-process plaintext, held in `SecretBuf`. |
| `KEKVersion` | key id version segment | the version GUID in the key identifier URL. |

`keyRef` is the key identifier: `https://VAULT.vault.azure.net/keys/KEYNAME` (version pinned or unversioned). For Managed HSM, host is `VAULT.managedhsm.azure.net`.

## 3. Context-binding caveat

`WrapKey`/`UnwrapKey` do not take AAD, so there is no native per-volume binding like AWS/GCP. Therefore:

- `BindsContext() bool { return false }`, so the conformance `ContextMismatchFails` test self-skips for this provider.
- To still bind the volume, prepend an authenticated header before wrapping: compute `tag = HMAC-SHA256(dek, volumeID)` and store `volumeID || tag || wrapped(dek)` in `Ciphertext`. On `Unwrap`, unwrap the DEK, recompute the HMAC over the stored `volumeID`, and verify it matches and that `volumeID` equals the requested one. This gives tamper-evident binding equivalent to AAD. Implement this as the default; document that it adds an HMAC the other providers get for free from the KMS.

## 4. Auth

Workload Identity (AKS) via `azidentity.NewDefaultAzureCredential` or `NewWorkloadIdentityCredential`. The federated identity needs `Key Wrap`/`Key Unwrap` permissions (access policy or RBAC role `Key Vault Crypto User`). Node identity gets wrap/unwrap only.

## 5. Config

```yaml
provider: azure-kv
keyRef: https://myvault.vault.azure.net/keys/topolvm-tde
wrapAlg: RSA-OAEP-256        # or A256KW for Managed HSM
bindVolume: true            # HMAC volume binding (Section 3)
```

## 6. Skeleton

```go
type azureProvider struct{ cli *azkeys.Client; keyName, keyVer, alg string; bind bool }

func (p *azureProvider) GenerateDEK(ctx context.Context, o keyprovider.KeyOpts) (keyprovider.SecretBuf, keyprovider.WrappedKey, error) {
	dek := crypt.NewSecretBuf(32)
	if _, err := rand.Read(dek.Bytes()); err != nil { return nil, keyprovider.WrappedKey{}, err }
	res, err := p.cli.WrapKey(ctx, p.keyName, p.keyVer, azkeys.KeyOperationParameters{
		Algorithm: to.Ptr(azkeys.EncryptionAlgorithm(p.alg)), Value: dek.Bytes(),
	}, nil)
	if err != nil { dek.Destroy(); return nil, keyprovider.WrappedKey{}, err }
	blob := p.encode(o.VolumeID, dek, res.Result)   // optional HMAC binding
	return dek, keyprovider.WrappedKey{Ciphertext: blob, KeyRef: p.keyRef(), KEKVersion: p.keyVer, Provider: "azure-kv"}, nil
}
// Unwrap -> decode blob, UnwrapKey, verify HMAC over volumeID, return SecretBuf.
// Rewrap -> Unwrap then WrapKey under current key version.
// KEKVersion -> p.keyVer (refresh from GetKey if unversioned keyRef).
func (p *azureProvider) BindsContext() bool { return p.bind }
```

## 7. Tests

`azurekv_test.go` (tag `kms_azure`): `providertest.Run`. CI uses a real test vault (Azure has no first-party local emulator for Key Vault crypto ops; gate behind tag + secrets, or use a recorded HTTP cassette via `azsdk` test proxy). Untagged unit tests cover the HMAC encode/decode and binding-failure path with the `fake` wrap backend.

## 8. Acceptance checklist

- [ ] `providertest.Run` green against a real test vault / Managed HSM.
- [ ] With `bindVolume: true`, unwrapping under a different volumeID fails the HMAC check (covered by a dedicated unit test since the shared context test self-skips).
- [ ] Rewrap preserves plaintext; in-process DEK held in `SecretBuf` and zeroized.
- [ ] Secret-leak grep clean.
